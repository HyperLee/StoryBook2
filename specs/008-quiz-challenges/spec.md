## Clarifications

### Session 2026-05-30

- Q: 問答挑戰的 canonical route 要固定為哪一個？ → A: 使用 `/quiz` 作為唯一 canonical route。
- Q: MVP 是否要顯示本次作答的累計分數或進度統計？ → A: 不顯示累計分數或進度統計，只顯示單題正誤、解釋與下一題動作。
- Q: 題目的「範圍」資料應如何建模？ → A: 題目只允許 `dinosaurs` 或 `aquarium` 來源範圍，`全部故事` 是聚合篩選。
- Q: 使用者完成目前範圍內最後一題後，選擇「下一題」時應如何處理？ → A: 循環回同範圍第一題。
- Q: 題目的 `difficulty` 允許值應該是哪些？ → A: 只允許 `easy` 與 `medium`。

## User Scenarios & Testing *(mandatory)*

### 使用者故事 1 - 進入問答挑戰並選擇範圍 (Priority: P1)

作為一位 5-10 歲孩童、陪讀家長或教師，我希望能從既有入口進入「問答挑戰」，並選擇全部故事、恐龍或水族館範圍，讓題目符合我想複習的內容。

**優先級理由**: 問答入口與範圍選擇是最小可用流程；若使用者不能開始答題，題庫與回饋都無法產生價值。

**獨立測試**: 從首頁或全站探索頁進入 `/quiz`，選擇「恐龍」範圍後，確認顯示一題與恐龍內容相關的選擇題。

**安全/資料影響**: 只讀取公開故事內容與專案維護題庫，不處理登入、個人資料、寫入型表單、外部內容查詢或作答紀錄保存。

**回滾方式**: 移除問答入口與問答頁後，既有恐龍、水族館、探索、比較、旅程與護照功能不受影響。

**Acceptance Scenarios**:

1. **Given** 使用者位於首頁，**When** 選擇「問答挑戰」，**Then** 系統導覽至 `/quiz`。
2. **Given** 使用者進入 `/quiz`，**When** 題庫可用，**Then** 系統顯示範圍選擇控制與開始答題動作。
3. **Given** 使用者選擇「全部故事」，**When** 開始答題，**Then** 系統可顯示恐龍或水族館相關題目。
4. **Given** 使用者選擇「恐龍」，**When** 開始答題，**Then** 系統只顯示恐龍範圍或恐龍相關題目。
5. **Given** 使用者選擇「水族館」，**When** 開始答題，**Then** 系統只顯示水族館範圍或水族館相關題目。

---

### 使用者故事 2 - 回答單題選擇題並得到友善回饋 (Priority: P1)

作為一位孩子，我希望回答一題選擇題後，馬上知道答案是否正確，並看到鼓勵式解釋，讓我能學會而不是被責備。

**優先級理由**: 作答與回饋是問答挑戰的核心互動，也是閱讀後理解檢核的主要價值。

**獨立測試**: 在問答挑戰頁選擇一題並作答，確認系統顯示正確或再試一次的友善回饋、解釋文字與下一題或再試一題動作。

**安全/資料影響**: 作答結果只存在目前頁面互動中，不建立可跨頁、跨裝置、重新載入或分享的作答紀錄。

**回滾方式**: 停用問答互動後，問答頁可退回可閱讀題目與相關故事連結；既有故事頁不受影響。

**Acceptance Scenarios**:

1. **Given** 使用者看到一題問答，**When** 選擇答案並提交，**Then** 系統顯示該答案是否正確。
2. **Given** 使用者答對，**When** 系統顯示回饋，**Then** 文字使用鼓勵語氣並提供簡短解釋。
3. **Given** 使用者答錯，**When** 系統顯示回饋，**Then** 文字不得羞辱、驚嚇或責備，並可提示再試或前往故事頁複習。
4. **Given** 使用者尚未選擇答案，**When** 嘗試提交，**Then** 系統顯示請先選一個答案的友善提示。
5. **Given** 使用者完成一題，**When** 選擇下一題或再試一題，**Then** 系統顯示同範圍內的下一個可用題目。

---

### 使用者故事 3 - 從題目回到相關故事複習 (Priority: P1)

作為一位使用者，我希望每題都能連回相關故事頁，讓我答錯或想知道更多時可以直接複習。

**優先級理由**: 問答不應脫離故事內容；相關故事連結能把測驗導回閱讀與探索。

**獨立測試**: 檢查任一題目，確認至少有一個相關故事連結，且該連結開啟既有恐龍或水族館故事詳情頁。

**安全/資料影響**: 相關故事連結只使用站內既有故事頁，不建立重複內容頁，不連往外部網站。

**回滾方式**: 移除題目中的複習連結後，問答仍可作答；既有故事詳情頁不受影響。

**Acceptance Scenarios**:

1. **Given** 題目與恐龍故事相關，**When** 使用者選擇「去讀故事」或等價動作，**Then** 系統開啟該恐龍既有故事詳情頁。
2. **Given** 題目與水族館故事相關，**When** 使用者選擇「去讀故事」或等價動作，**Then** 系統開啟該水族館既有故事詳情頁。
3. **Given** 題目有多個相關故事，**When** 系統顯示複習連結，**Then** 每個連結都必須有清楚名稱與來源標籤。
4. **Given** 題目引用不存在的故事項目，**When** 系統準備可顯示題目，**Then** 系統不得產生失效連結。

---

### 使用者故事 4 - 題庫資料完整並可降級 (Priority: P1)

作為一位使用者，我希望題目、選項、答案與解釋都完整清楚；如果題庫暫時無法讀取，也能看到友善狀態。

**優先級理由**: 題庫錯誤會直接造成錯誤學習，因此資料完整性與降級是 P1 必備。

**獨立測試**: 模擬題目缺少正確答案、選項不足或相關故事不存在，確認該題不會顯示；模擬題庫不可用，確認問答頁顯示友善狀態與回到既有入口的動作。

**安全/資料影響**: 錯誤訊息不得暴露檔案路徑、例外細節、設定內容、secret 或內部實作資訊。

**回滾方式**: 移除題庫與問答頁後，既有故事內容驗證與故事頁不受影響。

**Acceptance Scenarios**:

1. **Given** 題目少於 2 個選項，**When** 系統準備題目，**Then** 系統將該題視為無效。
2. **Given** 題目有超過 4 個選項，**When** 系統準備題目，**Then** 系統將該題視為不符合 MVP 範圍。
3. **Given** 題目沒有有效正確答案，**When** 系統準備題目，**Then** 系統不得向使用者顯示該題。
4. **Given** 題庫暫時不可用，**When** 使用者進入問答挑戰頁，**Then** 系統顯示問答暫時不能開始的友善狀態與返回既有入口的動作。
5. **Given** 資料載入失敗，**When** 系統顯示錯誤狀態，**Then** 頁面不得顯示內部例外文字、檔案路徑或診斷細節。

---

### 使用者故事 5 - 保持雙語、主題與可及性一致 (Priority: P2)

作為一位使用者，我希望問答挑戰和既有故事書一樣支援繁體中文與英文，並套用目前主題，讓我能用熟悉的閱讀方式答題。

**優先級理由**: 問答挑戰是跨故事書功能，必須遵守既有語言、主題與可及性規則。

**獨立測試**: 切換語言或主題後進入問答挑戰頁，確認範圍選擇、題目、選項、回饋、解釋、提示訊息、焦點狀態與操作目標都符合目前語言與有效主題，且問答頁不提供新的主題選擇控制項。

**安全/資料影響**: 語言與主題沿用既有網站偏好；問答頁不得新增每頁獨立主題設定、帳號同步、個人化保存或新的資料保存。

**回滾方式**: 移除問答專用雙語與樣式後，頁面可退回共用網站體驗；既有語言切換與主題規則不受影響。

**Acceptance Scenarios**:

1. **Given** 使用者已選擇英文，**When** 進入問答挑戰頁，**Then** 問答頁主要文字、題目、選項、回饋與解釋以英文顯示。
2. **Given** 使用者已選擇繁體中文，**When** 進入問答挑戰頁，**Then** 問答頁主要文字、題目、選項、回饋與解釋以繁體中文顯示。
3. **Given** 使用者已選擇深色模式，**When** 進入問答挑戰頁，**Then** 頁面套用深色外觀且不顯示新的主題選擇控制項。
4. **Given** 某題目欄位缺少所選語言內容，**When** 系統顯示題目或回饋，**Then** 系統回退繁體中文且不得顯示空白題目、空白選項或空白解釋。
5. **Given** 使用者使用鍵盤操作問答頁，**When** 焦點移動到範圍選擇、答案選項、提交、下一題或故事連結，**Then** 焦點清楚可見且控制項可被啟用。

### Edge Cases

- 題庫沒有任何有效題目時，系統顯示友善空狀態與返回首頁或故事入口的動作。
- 使用者選擇的範圍沒有有效題目時，系統顯示該範圍暫時沒有題目的友善提示。
- 題目缺少題幹、選項、正確答案、回饋、解釋或相關故事時，不得顯示給使用者。
- 題目選項少於 2 個或多於 4 個時，不符合 MVP 題型範圍。
- 題目有多個正確答案時，不符合單選題 MVP 範圍。
- 題目引用不存在的來源類型或故事識別時，不得產生失效複習連結。
- 題目 difficulty 不是 `easy` 或 `medium` 時，不符合 MVP 題庫範圍。
- 使用者尚未選擇答案就提交時，系統顯示友善提示，不得把未選視為答錯。
- 使用者快速連續點擊答案、提交或下一題時，畫面必須反映最後一次有效操作，不顯示互相矛盾的回饋。
- 使用者完成目前範圍內最後一題並選擇下一題時，系統循環回同範圍第一題。
- 語言偏好無效、遺失或翻譯缺漏時，系統回退繁體中文。
- 主題偏好無效、遺失或無法判斷時，頁面依既有整站主題規則採用安全外觀。
- 問答結果不得在重新載入、跨頁、跨裝置或可分享位置中保留。

## Requirements *(mandatory)*

### 範圍邊界

- 本階段包含問答入口、canonical route `/quiz`、範圍選擇、單題選擇題、作答提交、友善回饋、解釋文字、下一題或再試一題動作、相關故事連結、題庫完整性檢查與友善錯誤狀態。
- 本階段只整合目前既有的恐龍故事書與水族館故事書內容。
- 本階段題目必須由專案維護者人工撰寫與審核，不得使用外部題庫、外部百科資料、即時翻譯或自動產生內容作為使用者可見題目來源。
- 本階段不包含登入、排行榜、限時競賽、作答歷史、分數保存、伺服器端學習紀錄、跨裝置同步、後台管理、使用者上傳內容或新故事書來源。
- 本階段不保存作答狀態；重新整理、重新進入或前往其他頁面後，先前作答狀態不得被還原。
- 本階段不顯示本次作答的累計分數、答對/答錯統計或題數進度；MVP 只顯示目前單題的正誤、解釋與下一題或再試一題動作。
- 題目資料的來源範圍只允許 `dinosaurs` 或 `aquarium`；`全部故事` 只作為範圍選擇時聚合所有有效題目的 UI 篩選，不作為題目資料值。
- 本階段不改變既有恐龍、水族館、探索、比較、旅程、護照、語言或主題規則。

### Functional Requirements

- **FR-001**: 系統必須在首頁或全站探索頁至少一處提供清楚可辨識的「問答挑戰」入口。
- **FR-002**: 系統必須在唯一 canonical route `/quiz` 提供可直接開啟的問答挑戰頁，作為此功能的主要入口。
- **FR-003**: MVP 完整資料情境下必須提供至少 12 題有效題目，其中恐龍與水族館範圍各至少 5 題。
- **FR-004**: 每題必須具有唯一且穩定的 question id。
- **FR-005**: 每題必須包含雙語題幹、2-4 個雙語選項、一個正確選項、雙語正確回饋、雙語錯誤回饋、雙語解釋、來源範圍、difficulty 與至少一個相關故事引用；difficulty 只允許 `easy` 或 `medium`。
- **FR-006**: 題目資料的來源範圍只允許 `dinosaurs` 或 `aquarium`；問答頁的範圍選擇必須至少包含全部故事、恐龍與水族館，其中全部故事聚合所有有效題目。
- **FR-007**: 使用者必須能選擇問答範圍並開始答題。
- **FR-008**: 系統必須一次只顯示一題主要題目，避免資訊過載。
- **FR-009**: 使用者必須能用滑鼠、觸控或鍵盤選擇一個答案。
- **FR-010**: 使用者提交答案後，系統必須顯示該答案是否正確。
- **FR-011**: 答對與答錯回饋都必須使用鼓勵、非羞辱、非驚嚇的語氣。
- **FR-012**: 作答後系統必須顯示簡短解釋，幫助使用者理解答案。
- **FR-013**: 每題必須提供至少一個前往相關既有故事詳情頁的連結。
- **FR-014**: 恐龍相關故事連結必須前往對應恐龍故事詳情頁。
- **FR-015**: 水族館相關故事連結必須前往對應水族館故事詳情頁。
- **FR-016**: 使用者必須能在完成一題後前往同範圍的下一題或再試一題；若目前題目是該範圍最後一題，下一題必須循環回同範圍第一題。
- **FR-017**: 題目順序必須穩定且可預測；不得使用個人化、追蹤或外部服務決定題目。
- **FR-017a**: MVP 不得顯示累計分數、答對/答錯統計或題數進度；回饋必須限於目前單題的正誤、解釋與下一題或再試一題動作。
- **FR-018**: 系統必須拒絕缺少題幹、選項、正確答案、回饋、解釋、來源範圍、difficulty 或相關故事的題目，也必須拒絕 difficulty 不是 `easy` 或 `medium` 的題目。
- **FR-019**: 系統必須驗證每個相關故事引用存在於既有故事內容中。
- **FR-020**: 問答頁必須支援既有網站語言偏好，並在偏好無效或內容缺漏時回退繁體中文。
- **FR-021**: 問答頁必須套用既有整站有效主題，但不得提供新的主題選擇控制項。
- **FR-022**: 主題變更只能影響問答頁視覺呈現與控制項狀態，不得改變題目內容、作答狀態、語言偏好、導覽歷史或頁面捲動位置。
- **FR-023**: 所有使用者可見提示、空狀態、確認文字與錯誤訊息必須使用 5-10 歲孩童能理解的友善文字。
- **FR-024**: 所有互動控制項必須可用滑鼠、觸控與鍵盤操作，並具備可理解的 accessible name 與可見焦點。
- **FR-025**: 問答頁在手機、平板與桌面代表寬度下不得出現水平溢出、文字重疊或控制項互相遮擋。
- **FR-026**: 問答頁不得顯示空白題目、空白選項、空白回饋、空白解釋、空白來源標籤或失效連結。
- **FR-027**: 題庫或既有故事內容暫時不可用時，系統必須顯示友善錯誤狀態，並提供返回首頁或故事入口的動作。
- **FR-028**: 問答挑戰不得要求使用者登入、不得搜尋外部網站、不得新增即時翻譯、不得新增外部內容來源。
- **FR-029**: 作答狀態必須只保留於目前問答頁互動中；系統不得讓答題結果在重新整理、重新進入、跨頁、跨裝置或可分享位置中保留。
- **FR-030**: 答案狀態不得只靠顏色表示，必須搭配文字或可理解的狀態提示。
- **FR-031**: 問答核心流程不得依賴拖曳、限時或需要精細手勢的互動。

### Key Entities *(include if feature involves data)*

- **問答題目**: 一筆人工維護題目，包含 question id、來源範圍、difficulty、雙語題幹、雙語選項、正確選項、雙語回饋、雙語解釋與相關故事引用；來源範圍只允許 `dinosaurs` 或 `aquarium`，difficulty 只允許 `easy` 或 `medium`。
- **答案選項**: 題目中的一個可選答案，包含 option id、雙語文字、排序值，以及是否為該題唯一正確答案的標記。
- **問答範圍**: 使用者選擇的題目集合，目前包含全部故事、恐龍與水族館；全部故事是聚合篩選，不是題目資料來源值。
- **作答狀態**: 目前頁面中使用者選擇的題目、選項與回饋狀態；只存在於問答互動生命週期中。
- **題庫快照**: 問答頁可使用的題目集合與來源狀態，包含有效題目、範圍統計、相關故事連結與友善提示。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在至少 20 次代表性人工可用性驗收或同等瀏覽器任務紀錄中，95% 的任務能在 5 秒內從首頁或全站探索頁找到並進入問答挑戰。
- **SC-002**: 完整資料情境下，使用者能選擇全部故事、恐龍或水族館範圍並看到一題有效題目。
- **SC-003**: 使用者提交答案後，正確或錯誤回饋與解釋在 1 秒內顯示。
- **SC-004**: 100% 有效題目都至少有一個可開啟的既有故事詳情頁連結。
- **SC-005**: 在繁體中文與英文下，問答頁不得出現空白題目、空白選項、空白回饋或空白解釋。
- **SC-006**: 在 375px、768px 與 1366px 代表寬度下，問答頁不得水平溢出或出現文字與控制項重疊。
- **SC-007**: 鍵盤操作驗收中，範圍選擇、答案選項、提交、下一題與故事連結 100% 可聚焦並可啟用。
- **SC-008**: 題庫或既有故事內容部分不可用時，問答頁顯示可用題目或友善錯誤狀態，不呈現空白頁。
- **SC-009**: 重新整理、重新進入或前往其他頁面後，問答功能不得還原先前作答結果。
- **SC-010**: 在至少 10 位代表性測試使用者或同等人工驗收任務中，90% 能在作答後說出該題解釋中的一個關鍵學習點。

## Assumptions

- 初始問答只使用既有恐龍與水族館內容，不新增第三本故事書。
- 題目由專案維護者人工撰寫與審核，不由外部服務或自動化內容來源產生。
- 問答分數與作答結果不保存；若未來要把答題成果接入護照徽章，需另行更新護照與問答規格。
- 預設語言為繁體中文，英文為第二語言。
- 問答頁只做站內內容複習，不搜尋外部網站或百科資料。
