# StoryBook2 商業邏輯擴充 Roadmap

**建立日期**: 2026-05-19  
**目標**: 根據目前恐龍故事書、水族館故事書、深色模式與全站探索功能，整理下一階段可實作的商業邏輯擴充方向。  
**範圍限制**: 本文件聚焦產品與學習體驗，不規劃資安架構升級、登入、資料庫、後台管理、外部百科 API、即時翻譯或使用者上傳內容。

## 1. 目前產品商業邏輯摘要

StoryBook2 目前是一個兒童自然故事書網站，核心使用者是 5-10 歲孩童、陪讀家長與課堂展示者。既有功能已形成以下產品能力：

- **內容故事書**: 恐龍 8 筆與水族館動物 15 筆，以固定順序、單一詳情頁、短介紹、小故事與插圖呈現。
- **雙語學習**: 以繁體中文為預設語言，英文作為第二語言，使用 `storybook.language` 保存偏好。
- **站內探索**: `/explore` 統整恐龍與水族館內容，支援跨故事書搜尋與分類篩選。
- **閱讀舒適度**: 首頁支援亮色、深色與跟隨系統主題，使用 `storybook.theme` 保存偏好。
- **兒童友善狀態**: 找不到內容、搜尋無結果、資料載入失敗與圖片失敗都以孩子可理解文字呈現。
- **可直接連結**: 每個內容頁都有穩定 canonical route，並保留一般 anchor 導覽與瀏覽器 history。

因此，下一階段最有價值的商業邏輯不是單純增加按鈕或美化樣式，而是把已存在的內容變成更有教育價值、陪讀價值與反覆使用價值的流程。

## 2. 擴充判斷原則

後續功能建議符合以下原則：

- **重用既有 catalog**: 優先從 `StoryBook/Data/dinosaurs.json`、`StoryBook/Data/aquarium.json` 與 `/explore` projection 延伸。
- **維持無登入與無資料庫**: 近期功能使用 Razor Pages、JSON、localStorage 與靜態圖片即可落地。
- **讓孩子能完成任務**: 功能應支援「找得到、讀得懂、做得完、願意再來」。
- **讓家長或教師能陪讀**: 新功能應提供可提問、可比較、可回顧、可列印或可課堂使用的價值。
- **不破壞既有規則**: 保持雙語 fallback、主題一致、友善錯誤狀態、一般 anchor link 與可及性要求。
- **可分階段交付**: 每個功能都應能拆成獨立 user story，先做 P1 主要流程，再補 P2/P3 增強。

## 4. 可實作擴充項目

### P1-1. 主題學習旅程

**商業價值**  
把現有內容重新策展成「可完成的學習路線」，提升孩子的探索目標感，也讓家長或教師能快速選擇一段 5-15 分鐘的閱讀活動。

**目標使用者**  
5-10 歲孩童、陪讀家長、自然課教師。

**核心流程**  
使用者從首頁或 `/explore` 進入「學習旅程」，選擇一條主題路線，例如「草食朋友」、「會游泳的朋友」、「白堊紀探險」、「北美洲發現之旅」。頁面顯示路線介紹、包含的故事朋友、建議閱讀順序與開始閱讀動作。每讀完一筆內容後，可回到旅程繼續下一站。

**資料需求**  
新增靜態 JSON，例如 `StoryBook/Data/learning-journeys.json`，包含：

- journey slug
- sort order
- 雙語標題與簡介
- item references，例如 `dinosaurs:triceratops`、`aquarium:clownfish`
- 雙語學習目標
- 建議年齡或閱讀時間

**實作切片**

- 新增 `LearningJourney` model、validator、catalog service。
- 新增 `/journeys` 列表頁與 `/journeys/{slug}` 詳情頁。
- 在首頁與 `/explore` 加入學習旅程入口。
- 詳情頁中的故事項目使用既有 canonical route，不建立重複內容頁。
- 使用 localStorage 可選地記錄目前旅程閱讀進度，但不需要帳號。

**驗收標準**

- 使用者能從首頁進入旅程列表。
- 每條旅程至少包含 3 筆既有故事內容。
- 旅程詳情頁不得顯示空白標題、空白摘要或失效連結。
- 切換語言後，旅程標題、簡介與學習目標同步切換或 fallback 繁體中文。
- 故事連結仍前往 `/dinosaurs/{slug}` 或 `/aquarium/{slug}`。

**風險與注意事項**

- 旅程資料引用的 source/slug 必須驗證存在，避免出現死連結。
- 不要把旅程做成 JavaScript-only router。
- 不要把旅程進度寫入 URL、cookie 或伺服器端。

**不做範圍**

- 不做個人化推薦。
- 不做帳號同步。
- 不做由使用者自訂旅程。

### P1-2. 小小探險護照

**商業價值**  
用閱讀完成狀態、貼紙與成就增加回訪動機。這能把一次性瀏覽變成可累積的學習體驗，但仍可用 localStorage 實作，不需要登入。

**目標使用者**  
孩子與陪讀家長。

**核心流程**  
孩子閱讀故事頁後，可以按「我讀完了」或在頁面底部完成閱讀確認。系統在同一瀏覽器記錄已讀項目，並在「探險護照」頁顯示已收集的恐龍、水族館動物與旅程徽章。

**資料需求**  
以 localStorage 保存非敏感狀態：

- storage key: `storybook.readingPassport`
- item ids: `dinosaurs:tyrannosaurus-rex`
- completed journey ids
- last updated timestamp

可新增靜態 badge metadata：

- badge code
- 雙語名稱與描述
- 取得條件，例如讀完 3 隻草食朋友

**實作切片**

- 新增護照 service 或 client-side module，集中讀寫 localStorage。
- 在恐龍、水族館與未來旅程頁加入完成閱讀控制。
- 新增 `/passport` 頁面顯示進度與徽章。
- 首頁加入「我的探險護照」入口。
- 提供清除本機護照資料的明確控制與確認文字。

**驗收標準**

- 完成閱讀後，重新整理同一頁仍顯示已完成狀態。
- `/passport` 顯示已讀數量、總數與已取得徽章。
- 無法讀取 localStorage 時，頁面仍可閱讀故事，只是不保存進度。
- 清除護照只刪除 `storybook.readingPassport`，不得影響 `storybook.language` 或 `storybook.theme`。

**風險與注意事項**

- 這不是跨裝置同步，文案必須清楚避免誤解。
- 完成閱讀應由孩子可理解的動作觸發，不要自動把打開頁面視為讀完。
- localStorage 資料不可包含姓名、年齡或其他個資。

**不做範圍**

- 不做登入會員。
- 不做排行榜。
- 不做伺服器端學習紀錄。

### P1-3. 內容比較器

**商業價值**  
比較是自然科學學習的核心能力。讓孩子比較兩個故事朋友，可以把既有欄位轉成更深層的學習互動，例如「三角龍和海龜有什麼不同？」、「暴龍和鯊魚都吃肉嗎？」。

**目標使用者**  
孩子、陪讀家長、教師。

**核心流程**  
使用者在 `/compare` 選擇兩個故事朋友，頁面顯示左右比較表，包括來源、名稱、分類、食性、生活時期或生活區域、發現地點、摘要與前往詳情頁動作。

**資料需求**  
可重用 `ExplorationItem` projection，但需要補足比較欄位：

- diet
- period
- living area
- discovery location
- source-specific note

若現有 `ExplorationItem` 不足，可新增 `ComparisonItem` projection service，由恐龍與水族館 catalog 組合產生。

**實作切片**

- 新增 `ComparisonCatalogService`，輸出可比較項目與欄位。
- 新增 `/compare` Razor Page。
- 頁面使用一般 select 或卡片選擇兩筆內容。
- 選擇狀態可保留在目前頁面，不需要 URL query；若要支援分享再另開 spec。
- 從 `/explore` 結果卡提供「加入比較」可作 P2。

**驗收標準**

- 使用者能選擇任兩筆不同內容並看到比較表。
- 恐龍與水族館欄位差異清楚顯示，不適用欄位以友善文字說明。
- 比較表中每筆內容都有前往原詳情頁的 anchor link。
- 雙語切換後，欄位名稱與內容同步切換或 fallback。
- 在 375px、768px、1366px 下比較表不水平溢出。

**風險與注意事項**

- 不要為了比較而改動既有 catalog 的核心內容格式。
- 不適用欄位需要兒童友善文字，例如「這本故事沒有生活時期欄位」。
- 比較器不能取代原故事詳情頁。

**不做範圍**

- 不做 AI 自動比較文字。
- 不做使用者儲存比較組合。
- 不做任意外部資料比較。

### P2-1. 互動問答挑戰

**商業價值**  
把閱讀後的理解檢核做成小遊戲，可以提升停留時間與學習成效。問答也適合課堂互動與親子陪讀收尾。

**目標使用者**  
孩子、家長、教師。

**核心流程**  
使用者進入「問答挑戰」，選擇範圍，例如全部故事、恐龍、水族館或某條學習旅程。系統顯示單題選擇題，孩子作答後得到友善回饋與「再試一題」或「去讀故事」動作。

**資料需求**  
建議新增靜態 JSON，例如 `StoryBook/Data/quiz-questions.json`：

- question id
- source scope 或 related item ids
- 雙語題目
- 2-4 個雙語選項
- correct option id
- 雙語解釋
- difficulty: easy/medium

**實作切片**

- 新增 quiz model、validator、catalog service。
- 新增 `/quiz` 頁面。
- 問題順序可先使用固定順序或 deterministic shuffle，不做個人化推薦。
- 作答狀態只存在目前頁面，分數可選擇保存在 localStorage 作為護照徽章條件。
- 每題提供「回故事頁複習」連結。

**驗收標準**

- 問題與選項在雙語下都不空白。
- 答錯時用鼓勵文字提示，不使用羞辱或驚嚇語氣。
- 每題至少有一個相關故事連結。
- 無 JavaScript 時仍應顯示可閱讀題目與回故事頁動作，互動評分可降級。
- 若題庫載入失敗，顯示友善狀態與回首頁。

**風險與注意事項**

- 自動從 catalog 生成題目容易產生語意不自然的題目，第一版建議人工維護題庫。
- 選項不可太長，避免兒童閱讀負擔。
- 答題結果不得記錄到伺服器。

**不做範圍**

- 不做排名。
- 不做限時競賽。
- 不做外部題庫。

### P2-2. 雙語自然詞彙卡

**商業價值**  
把既有中英文內容轉成詞彙學習工具，擴大產品在英語學習、親子共讀與課堂展示的用途。

**目標使用者**  
雙語學習中的孩子、家長、教師。

**核心流程**  
使用者進入「自然詞彙卡」，依分類查看詞彙，例如食性、生活區域、時期、動物名稱。每張卡顯示中文、英文、簡短例句與相關故事連結。

**資料需求**  
可先從既有 catalog 萃取：

- 名稱: 暴龍 / Tyrannosaurus Rex
- 食性: 草食性 / Herbivore
- 生活區域: 珊瑚礁 / Coral reef
- 生活時期: 白堊紀晚期 / Late Cretaceous

若要提供例句，建議新增 `StoryBook/Data/vocabulary-cards.json`。

**實作切片**

- 新增 vocabulary projection service，從恐龍、水族館與 habitat categories 建立詞彙集合。
- 新增 `/vocabulary` 頁面。
- 支援依詞彙類型篩選。
- 詞彙卡連回相關故事頁或 `/explore` 篩選結果。
- 可加入「翻面」效果，但不可讓核心內容只依賴動畫。

**驗收標準**

- 詞彙卡不重複顯示同一組中英文詞。
- 每張卡至少有一個相關故事連結或相關內容數量。
- 語言切換不造成卡片空白。
- 搜尋或篩選狀態不寫入 URL 或伺服器。
- 鍵盤可操作所有卡片控制。

**風險與注意事項**

- 英文詞彙大小寫與複數形式要一致。
- 中文詞彙應維持繁體中文。
- 不要用即時翻譯生成詞彙。

**不做範圍**

- 不做語音朗讀。
- 不做拼字測驗。
- 不做外部字典查詢。

### P2-3. 家長與教師陪讀指南

**商業價值**  
提供成人可直接使用的提問、活動與學習目標，能讓網站從孩子自行閱讀延伸到親子共讀與課堂使用，提升實際使用場景。

**目標使用者**  
陪讀家長、國小教師、課後活動帶領者。

**核心流程**  
每個故事詳情頁或旅程頁提供「陪讀指南」區塊，包含閱讀前提問、閱讀後討論、觀察重點與延伸活動。也可新增 `/guides` 彙整全部指南。

**資料需求**  
建議在各 catalog 或獨立 guide JSON 中新增：

- related item id
- 雙語 learning goals
- 雙語 discussion questions
- estimated reading time
- suggested activity

**實作切片**

- 第一階段只在詳情頁底部顯示指南，不新增獨立 route。
- 第二階段新增 `/guides` 給成人快速查找。
- 指南內容由 JSON 驗證，不允許空白問題或過長文字。
- 在列印活動單功能前，先確保指南內容穩定。

**驗收標準**

- 每筆內容至少有 2 個提問與 1 個延伸活動。
- 指南文字明確標示給成人使用，不干擾孩子主要閱讀流程。
- 雙語切換後指南同步切換或 fallback。
- 若某筆指南缺漏，頁面不顯示空白指南區塊。

**風險與注意事項**

- 成人指南文字可以比兒童故事長，但仍應簡潔。
- 不要把指南放在頁面最前面，避免干擾孩子主要內容。
- 不要引入教學成效追蹤或個資蒐集。

**不做範圍**

- 不做班級管理。
- 不做學生作業繳交。
- 不做教師帳號。

### P3-1. 新故事書內容包

**商業價值**  
新增第三本故事書能擴大內容宇宙，讓 `/explore`、旅程、比較器、詞彙卡與問答挑戰都有更多資料來源。

**候選內容包**

- **昆蟲故事書**: 蝴蝶、蜜蜂、瓢蟲、螞蟻、蜻蜓、螢火蟲等。
- **植物故事書**: 向日葵、樹、仙人掌、水生植物、蕨類等。
- **太空故事書**: 太陽、月亮、地球、火星、土星、彗星等。
- **天氣故事書**: 雲、雨、彩虹、風、雪、雷等。

**推薦首選**  
昆蟲故事書。理由是它與目前自然朋友定位一致，容易和恐龍/水族館做比較，也適合孩子日常觀察。

**核心流程**  
首頁新增第三入口，使用者進入新故事書主頁，逐頁閱讀每個主題內容。新內容也要進入 `/explore`、旅程、比較器與問答挑戰。

**資料需求**

- 新 catalog JSON，例如 `StoryBook/Data/insects.json`
- 新圖片目錄，例如 `StoryBook/wwwroot/images/insects/`
- 新 source type，例如 `insects`
- 雙語名稱、分類、棲地、食物或特徵、摘要、小故事、圖片 alt

**實作切片**

- 先用 Spec Kit 建立完整 spec、plan、tasks。
- 複製既有 Dinosaur/Aquarium 的 catalog service 模式，但不要盲目複製不適用欄位。
- 擴充 `ExplorationSourceType` 與 `ExplorationCatalogService`。
- 補足整合測試，確保新來源加入 `/explore`。
- 最後再補旅程、問答與比較器整合。

**驗收標準**

- 新故事書至少 8-12 筆完整雙語內容。
- 每筆內容有穩定 slug 與 canonical route。
- `/explore` 能搜尋、篩選並連回新詳情頁。
- 內容長度、圖片 alt、找不到狀態與雙語 fallback 符合現有規則。

**風險與注意事項**

- 最大成本是內容與圖片，不是程式碼。
- 新 source type 會影響探索、測試與未來功能，需完整規劃。
- 不要一次同時新增多本故事書。

**不做範圍**

- 不做外部百科資料同步。
- 不做 CMS。
- 不做使用者投稿。

### P3-2. 列印活動單

**商業價值**  
讓網站內容能進入離線親子活動與課堂活動。列印活動單可提高教師使用率，也能讓孩子離開螢幕後繼續學習。

**目標使用者**  
家長、教師、課後活動帶領者。

**核心流程**  
使用者在旅程、詞彙卡或問答頁選擇「列印活動單」，看到適合列印的頁面，包含故事朋友清單、觀察問題、簡單配對題與空白繪圖區。

**資料需求**  
可重用旅程、詞彙卡與問答資料。若需要固定版型，可新增 print template metadata：

- worksheet type
- related journey or source
- 雙語標題
- activity blocks

**實作切片**

- 新增 print-specific Razor Page，例如 `/print/journeys/{slug}`。
- 新增 print CSS，只處理列印版面。
- 第一版只支援旅程活動單。
- 活動單使用一般 HTML，不生成 PDF。

**驗收標準**

- 瀏覽器列印預覽中內容不被截斷。
- 活動單包含標題、學習目標、故事清單與至少 2 個活動題。
- 不列印導覽列、主題 selector 或互動按鈕。
- 雙語切換後列印頁內容同步切換或 fallback。

**風險與注意事項**

- 列印 CSS 需要手動驗收。
- 不要加入伺服器端 PDF 生成依賴，除非未來有明確需求。
- 活動單內容需避免過滿，適合 A4 或 Letter。

**不做範圍**

- 不做 PDF 下載。
- 不做學生姓名欄位保存。
- 不做作業提交。

### P3-3. 今日探索推薦

**商業價值**  
在不做個人化與帳號的前提下，提供每日或每次進站的推薦主題，降低使用者不知道從哪裡開始的問題。

**目標使用者**  
第一次造訪的孩子與家長。

**核心流程**  
首頁顯示「今天可以看看」區塊，推薦一筆故事朋友、一條旅程或一組比較題。推薦可以由日期 deterministic 選出，不需要追蹤使用者。

**資料需求**  
可重用既有 catalog 與旅程資料。推薦規則可以是：

- 依日期對可用項目數取餘數
- 依 source 輪替
- 避免連續只推薦同一故事書

**實作切片**

- 新增 `DailyRecommendationService`。
- 首頁 PageModel 取得推薦資料。
- 首頁新增推薦區塊與一般 anchor link。
- 測試固定日期下輸出穩定推薦。

**驗收標準**

- 首頁至少顯示一個可點擊推薦。
- 推薦連結前往既有頁面或旅程頁。
- 所有 catalog 載入失敗時，首頁顯示友善 fallback。
- 不使用 localStorage、cookie 或伺服器狀態保存推薦。

**風險與注意事項**

- 推薦規則要簡單可解釋。
- 不要宣稱個人化。
- 不要因推薦區塊讓首頁主入口變難找。

**不做範圍**

- 不做 AI 推薦。
- 不做使用者行為追蹤。
- 不做後台配置。

### P3-4. 自然朋友關係地圖

**商業價值**  
用視覺方式呈現故事朋友之間的關係，例如同食性、同地點、同棲地或同時期。這能把分類搜尋進一步升級成探索網路。

**目標使用者**  
喜歡視覺探索的孩子、教師。

**核心流程**  
使用者進入「關係地圖」，選擇一個故事朋友後，看到和它有共同特徵的其他故事朋友。每個關係都有簡短原因，例如「都吃植物」、「都住在珊瑚礁附近」、「都在北美洲被發現」。

**資料需求**  
可由 catalog facets 推導：

- source
- diet
- living area
- period
- discovery location

若需要更自然的文案，可新增 relationship labels。

**實作切片**

- 新增 relationship projection service。
- 新增 `/relationships` 頁面。
- 初版使用卡片與連線說明，不需要 canvas 或複雜圖形。
- 從詳情頁加入「看看相似朋友」入口。

**驗收標準**

- 任一故事朋友至少顯示 0 個或多個相關項目；0 個時有友善提示。
- 每個相關項目都說明共同特徵。
- 點擊相關項目開啟既有詳情頁。
- 版面在 mobile/tablet/desktop 不重疊。

**風險與注意事項**

- 自動關係可能過多，需要限制數量。
- 共同特徵要兒童可理解，不要顯示內部 facet code。
- 初版不做複雜力導向圖，降低可及性風險。

**不做範圍**

- 不做 3D 圖。
- 不做拖曳式心智圖。
- 不做外部知識圖譜。

## 5. 建議實作順序

建議以以下順序推進，讓每一步都能單獨交付，也能替下一步累積資料與 UI 模式：

1. **主題學習旅程**: 先建立策展內容與跨故事書閱讀流程。
2. **小小探險護照**: 在旅程和故事頁加入完成感與回訪動機。
3. **內容比較器**: 擴充探索深度，重用已建立的 projection 思路。
4. **互動問答挑戰**: 用旅程與故事內容做閱讀後活動。
5. **雙語自然詞彙卡**: 補強英文學習與課堂展示用途。
6. **家長/教師陪讀指南**: 增加成人使用價值，支援後續列印活動單。
7. **列印活動單**: 把旅程、詞彙與問答轉成線下活動。
8. **新故事書內容包**: 在互動模式穩定後，再擴大內容來源。

## 6. 近期 Top 3 功能的最低可行版本

### Top 1: 主題學習旅程 MVP

- 3 條旅程，每條 3-5 筆既有內容。
- `/journeys` 與 `/journeys/{slug}`。
- 雙語標題、簡介與學習目標。
- 從旅程連到既有詳情頁。
- 不記錄進度，先驗證策展價值。

### Top 2: 小小探險護照 MVP

- 只記錄已讀故事項目。
- 每個詳情頁加入「我讀完了」控制。
- `/passport` 顯示已讀數量與清單。
- 使用 localStorage，不做登入。
- 提供清除本機護照資料。

### Top 3: 內容比較器 MVP

- `/compare` 選擇兩個故事朋友。
- 顯示來源、名稱、食性、生活區域/生活時期、發現地點與摘要。
- 不適用欄位顯示友善文字。
- 可從比較表回到原詳情頁。
- 不保存比較狀態。

## 7. 不建議近期投入的方向

以下項目可能有價值，但不適合目前階段：

- **登入與跨裝置同步**: 會引入帳號、資料保護、授權與資料庫，超出目前產品型態。
- **後台 CMS**: 目前內容量仍可用 JSON 與 Spec Kit 管理，CMS 會增加維護成本。
- **外部百科 API**: 會破壞已審核、兒童友善、雙語固定內容的品質控制。
- **即時翻譯**: 不適合 5-10 歲教育內容，且會造成翻譯品質與 fallback 風險。
- **AI 自動問答或推薦**: 目前內容規模小，人工題庫與 deterministic 推薦更穩定。
- **社群分享與留言**: 目標使用者是兒童，不應在沒有完整安全與審核策略前引入互動社群。

## 8. 後續 Spec Kit 建議

若要正式實作，建議依序建立下列 feature specs：

1. `005-learning-journeys`: 主題學習旅程。
2. `006-reading-passport`: 小小探險護照。
3. `007-story-friend-compare`: 內容比較器。
4. `008-quiz-challenges`: 互動問答挑戰。
5. `009-vocabulary-cards`: 雙語自然詞彙卡。

每份 spec 都應維持現有專案規則：

- 使用繁體中文撰寫使用者故事與驗收情境。
- 先定義測試與驗收，再實作。
- Razor Pages + services + JSON data。
- 不新增資料庫、登入、外部 API 或 JavaScript-only routing。
- 對齊 `storybook.language`、`storybook.theme` 與既有可及性規則。
