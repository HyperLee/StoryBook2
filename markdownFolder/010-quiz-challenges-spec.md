# 功能規格: 互動問答挑戰

**Feature Branch**: `010-quiz-challenges`

**Created**: 2026-05-30

**Status**: Draft

**Input**: User description: "根據既有 001-005 規格與目前專案，新增下一階段可擴充功能規格至 markdownFolder。選定項目為學習閉環中的互動問答挑戰 MVP。"

## 功能目標

建立「互動問答挑戰」功能，讓孩子、陪讀家長與教師可以在 `/quiz` 選擇問答範圍，回答與既有恐龍故事書與水族館故事書相關的人工維護選擇題。每題作答後，系統顯示鼓勵式回饋、簡短解釋與回到相關故事頁複習的連結。

本功能使用靜態人工題庫，不從外部題庫、百科 API、即時翻譯或 AI 自動生成題目。作答狀態只存在目前頁面互動中，不新增登入、資料庫、排行榜、限時競賽、伺服器端成績紀錄、跨裝置同步或使用者個人資料蒐集。

## 選擇此項目的理由

互動問答挑戰能把閱讀後的理解檢核變成簡短遊戲，原因如下：

- 能把既有故事內容轉成「讀完後做一題」的學習閉環。
- 適合親子共讀收尾、課堂展示與孩子自我複習。
- 人工題庫能維持兒童友善、語意自然與雙語品質。
- 可重用現有 catalog、詳情頁 route、語言偏好、主題與錯誤狀態規則。
- 不需要先做登入、資料庫、排名或外部服務。

## 使用者情境與測試 *(mandatory)*

### 使用者故事 1 - 進入問答挑戰並選擇範圍 (Priority: P1)

作為一位孩子、陪讀家長或教師，我希望能從首頁或探索頁進入問答挑戰，並選擇全部故事、恐龍或水族館範圍，讓題目符合我想複習的內容。

**優先級理由**: 問答入口與範圍選擇是最小可用流程；若使用者不能開始答題，題庫與回饋都無法產生價值。

**獨立測試**: 從首頁或 `/explore` 進入 `/quiz`，選擇「恐龍」範圍後，確認顯示一題與恐龍內容相關的選擇題。

**安全/資料影響**: 只讀取公開靜態題庫與既有 catalog，不處理登入、個人資料、寫入型表單、外部 API 或伺服器端作答紀錄。

**回滾方式**: 移除首頁與 `/explore` 的問答入口，以及 `/quiz` Razor Page route；既有恐龍、水族館、探索、比較、旅程與護照功能不受影響。

**驗收情境**:

1. **Given** 使用者位於首頁，**When** 選擇「問答挑戰」，**Then** 系統導覽至 `/quiz`。
2. **Given** 使用者進入 `/quiz`，**When** 題庫載入完成，**Then** 系統顯示範圍選擇控制與開始答題動作。
3. **Given** 使用者選擇「全部故事」，**When** 開始答題，**Then** 系統可顯示恐龍或水族館相關題目。
4. **Given** 使用者選擇「恐龍」，**When** 開始答題，**Then** 系統只顯示恐龍範圍或恐龍相關題目。
5. **Given** 使用者選擇「水族館」，**When** 開始答題，**Then** 系統只顯示水族館範圍或水族館相關題目。

---

### 使用者故事 2 - 回答單題選擇題並得到友善回饋 (Priority: P1)

作為一位孩子，我希望回答一題選擇題後，馬上知道答案是否正確，並看到鼓勵式解釋，讓我能學會而不是被責備。

**優先級理由**: 作答與回饋是問答挑戰的核心互動，也是閱讀後理解檢核的主要價值。

**獨立測試**: 在 `/quiz` 選擇一題並作答，確認系統顯示正確或再試一次的友善回饋、解釋文字與「下一題」或「再試一題」動作。

**安全/資料影響**: 作答結果只存在目前頁面，不寫入 URL、localStorage、sessionStorage、cookie、伺服器端狀態或外部服務。

**回滾方式**: 停用問答互動 script 後，`/quiz` 可退回可閱讀題目與相關故事連結；既有故事頁不受影響。

**驗收情境**:

1. **Given** 使用者看到一題問答，**When** 選擇答案並提交，**Then** 系統顯示該答案是否正確。
2. **Given** 使用者答對，**When** 系統顯示回饋，**Then** 文字使用鼓勵語氣並提供簡短解釋。
3. **Given** 使用者答錯，**When** 系統顯示回饋，**Then** 文字不得羞辱、驚嚇或責備，並可提示再試或前往故事頁複習。
4. **Given** 使用者尚未選擇答案，**When** 嘗試提交，**Then** 系統顯示請先選一個答案的友善提示。
5. **Given** 使用者完成一題，**When** 選擇下一題或再試一題，**Then** 系統顯示同範圍內的下一個可用題目。

---

### 使用者故事 3 - 從題目回到相關故事複習 (Priority: P1)

作為一位使用者，我希望每題都能連回相關故事頁，讓我答錯或想知道更多時可以直接複習。

**優先級理由**: 問答不應脫離故事內容；相關故事連結能把測驗導回閱讀與探索。

**獨立測試**: 檢查任一題目，確認至少有一個相關故事連結，且該連結開啟既有 `/dinosaurs/{slug}` 或 `/aquarium/{slug}` 詳情頁。

**安全/資料影響**: 相關故事連結只使用既有 canonical route，不建立重複內容頁，不連外。

**回滾方式**: 移除題目頁的複習連結後，問答仍可作答；既有故事詳情頁不受影響。

**驗收情境**:

1. **Given** 題目與恐龍故事相關，**When** 使用者選擇「去讀故事」或等價動作，**Then** 系統使用一般 anchor link 開啟 `/dinosaurs/{slug}`。
2. **Given** 題目與水族館故事相關，**When** 使用者選擇「去讀故事」或等價動作，**Then** 系統使用一般 anchor link 開啟 `/aquarium/{slug}`。
3. **Given** 題目有多個相關故事，**When** 系統顯示複習連結，**Then** 每個連結都必須有清楚名稱與來源標籤。
4. **Given** 題目引用不存在的故事項目，**When** 系統建立題目 snapshot，**Then** 系統不得產生失效連結。

---

### 使用者故事 4 - 題庫資料完整並可降級 (Priority: P1)

作為一位使用者，我希望題目、選項、答案與解釋都完整清楚；如果題庫暫時無法讀取，也能看到友善狀態。

**優先級理由**: 題庫錯誤會直接造成錯誤學習，因此資料驗證與降級是 P1 必備。

**獨立測試**: 模擬題目缺少正確答案、選項不足或相關故事不存在，確認 validator 能偵測錯誤；模擬題庫載入失敗，確認 `/quiz` 顯示友善狀態與回首頁動作。

**安全/資料影響**: 錯誤訊息不得暴露檔案路徑、例外 stack trace、設定內容、secret 或內部實作細節。

**回滾方式**: 移除題庫與問答頁後，既有 catalog 驗證與故事頁不受影響。

**驗收情境**:

1. **Given** 題目少於 2 個選項，**When** 題庫驗證執行，**Then** 系統將該題視為無效。
2. **Given** 題目有超過 4 個選項，**When** 題庫驗證執行，**Then** 系統將該題視為不符合 MVP 範圍。
3. **Given** 題目沒有有效正確答案，**When** 題庫驗證執行，**Then** 系統不得向使用者顯示該題。
4. **Given** 題庫載入失敗，**When** 使用者進入 `/quiz`，**Then** 系統顯示問答暫時不能開始的友善狀態與回首頁動作。
5. **Given** 資料載入失敗，**When** 系統顯示錯誤狀態，**Then** 頁面不得顯示 `System.IO`、檔案路徑、stack trace 或內部例外文字。

---

### 使用者故事 5 - 保持雙語、主題與可及性一致 (Priority: P2)

作為一位使用者，我希望問答挑戰和既有故事書一樣支援繁體中文與英文，並套用目前主題，讓我能用熟悉的閱讀方式答題。

**優先級理由**: 問答挑戰是跨故事書功能，必須遵守既有語言、主題與可及性規則。

**獨立測試**: 切換語言或主題後進入 `/quiz`，確認範圍選擇、題目、選項、回饋、解釋、提示訊息、焦點狀態與操作目標都符合目前語言與有效主題，且問答頁不提供新的主題 selector。

**安全/資料影響**: 語言與主題沿用既有 `localStorage` 偏好；問答頁不得新增 theme selector、server-side preference endpoint、cookie、帳號同步或新的資料保存。

**回滾方式**: 移除問答專用雙語與樣式後，頁面可退回共用 layout；既有語言切換與主題服務不受影響。

**驗收情境**:

1. **Given** 使用者已選擇英文，**When** 進入 `/quiz`，**Then** 問答頁主要文字、題目、選項、回饋與解釋以英文顯示。
2. **Given** 使用者已選擇繁體中文，**When** 進入 `/quiz`，**Then** 問答頁主要文字、題目、選項、回饋與解釋以繁體中文顯示。
3. **Given** 使用者已選擇深色模式，**When** 進入 `/quiz`，**Then** 頁面套用深色外觀且不顯示新的主題選擇控制項。
4. **Given** 某題目欄位缺少所選語言內容，**When** 系統顯示題目或回饋，**Then** 系統回退繁體中文且不得顯示空白題目、空白選項或空白解釋。
5. **Given** 使用者使用鍵盤操作問答頁，**When** 焦點移動到範圍選擇、答案選項、提交、下一題或故事連結，**Then** 焦點清楚可見且控制項可被啟用。

## 邊界情況

- 題庫沒有任何有效題目時，系統顯示友善空狀態與返回首頁或故事入口的動作。
- 使用者選擇的範圍沒有有效題目時，系統顯示該範圍暫時沒有題目的友善提示。
- 題目缺少題幹、選項、正確答案、回饋、解釋或相關故事時，不得顯示給使用者。
- 題目選項少於 2 個或多於 4 個時，不符合 MVP 題型範圍。
- 題目有多個正確答案時，不符合單選題 MVP 範圍。
- 題目引用不存在的 source type 或 slug 時，不得產生失效複習連結。
- 使用者尚未選擇答案就提交時，系統顯示友善提示，不得把未選視為答錯。
- 使用者快速連續點擊答案、提交或下一題時，畫面必須反映最後一次有效操作，不顯示互相矛盾的回饋。
- 語言偏好無效、遺失或翻譯缺漏時，系統回退繁體中文。
- 主題偏好無效、遺失或無法判斷時，頁面依既有整站主題規則採用安全外觀。
- 問答結果不得保存到 localStorage、sessionStorage、cookie、URL query 或伺服器端狀態。

## 需求 *(mandatory)*

### 範圍邊界

- 本階段包含首頁或 `/explore` 的問答入口、canonical route `/quiz`、範圍選擇、單題選擇題、作答提交、友善回饋、解釋文字、下一題或再試一題動作、相關故事連結、題庫驗證與友善錯誤狀態。
- 本階段只整合目前既有的恐龍故事書與水族館故事書內容。
- 本階段可新增靜態題庫檔，例如 `StoryBook/Data/quiz-questions.json`，但不得改動 `StoryBook/Data/dinosaurs.json` 或 `StoryBook/Data/aquarium.json` 的既有 schema。
- 本階段題目必須人工維護，不得使用 AI、外部百科 API、外部題庫或即時翻譯自動產生使用者可見題目。
- 本階段不包含登入、排行榜、限時競賽、作答歷史、分數保存、伺服器端學習紀錄、跨裝置同步、後台管理、使用者上傳內容或新故事書來源。
- 本階段不將作答狀態寫入 URL query、browser history、localStorage、sessionStorage、cookie、伺服器端狀態或任何外部服務。
- 本階段不改變既有恐龍、水族館、`/explore`、`/compare`、語言或主題規則。

### 功能需求

- **FR-001**: 系統必須在首頁或 `/explore` 至少一處提供清楚可辨識的「問答挑戰」入口。
- **FR-002**: 系統必須在 canonical route `/quiz` 提供問答挑戰頁面。
- **FR-003**: MVP 完整資料情境下必須提供至少 12 題有效題目，其中恐龍與水族館範圍各至少 5 題。
- **FR-004**: 每題必須具有唯一且穩定的 question id。
- **FR-005**: 每題必須包含雙語題幹、2-4 個雙語選項、一個正確選項、雙語正確回饋、雙語錯誤回饋、雙語解釋、範圍、難度與至少一個相關故事引用。
- **FR-006**: 題目範圍必須至少包含全部故事、恐龍與水族館。
- **FR-007**: 使用者必須能選擇問答範圍並開始答題。
- **FR-008**: 系統必須一次只顯示一題主要題目，避免資訊過載。
- **FR-009**: 使用者必須能用滑鼠、觸控或鍵盤選擇一個答案。
- **FR-010**: 使用者提交答案後，系統必須顯示該答案是否正確。
- **FR-011**: 答對與答錯回饋都必須使用鼓勵、非羞辱、非驚嚇的語氣。
- **FR-012**: 作答後系統必須顯示簡短解釋，幫助使用者理解答案。
- **FR-013**: 每題必須提供至少一個前往相關既有故事詳情頁的 anchor link。
- **FR-014**: 恐龍相關故事連結必須前往 `/dinosaurs/{slug}`。
- **FR-015**: 水族館相關故事連結必須前往 `/aquarium/{slug}`。
- **FR-016**: 使用者必須能在完成一題後前往同範圍的下一題或再試一題。
- **FR-017**: 問題順序可以使用固定順序或 deterministic shuffle，但不得使用個人化、追蹤或外部服務。
- **FR-018**: 題庫 validator 必須拒絕缺少題幹、選項、正確答案、回饋、解釋、範圍或相關故事的題目。
- **FR-019**: 題庫 validator 必須驗證每個相關故事引用存在於既有 catalog。
- **FR-020**: 問答頁必須支援既有網站語言偏好，並在偏好無效或內容缺漏時回退繁體中文。
- **FR-021**: 問答頁必須套用既有整站有效主題，但不得提供新的主題選擇控制項。
- **FR-022**: 主題變更只能影響問答頁視覺呈現與控制項狀態，不得改變題目內容、作答狀態、語言偏好、導覽歷史或頁面捲動位置。
- **FR-023**: 所有使用者可見提示、空狀態、確認文字與錯誤訊息必須使用 5-10 歲孩童能理解的友善文字。
- **FR-024**: 所有互動控制項必須可用滑鼠、觸控與鍵盤操作，並具備可理解的 accessible name 與可見焦點。
- **FR-025**: 問答頁在手機、平板與桌面代表寬度下不得出現水平溢出、文字重疊或控制項互相遮擋。
- **FR-026**: 問答頁不得顯示空白題目、空白選項、空白回饋、空白解釋、空白來源標籤或失效連結。
- **FR-027**: 題庫或 catalog 載入失敗時，系統必須顯示友善錯誤狀態，並提供返回首頁或故事入口的動作。
- **FR-028**: 問答挑戰不得要求使用者登入、不得搜尋外部網站、不得新增即時翻譯、不得新增外部內容來源。
- **FR-029**: 作答狀態必須只保留於目前 `/quiz` 頁面互動中；系統不得把答題結果寫入 URL query、瀏覽器 history、localStorage、sessionStorage、cookie 或伺服器端狀態。

### 設計需求

- **DR-001**: 問答頁視覺風格必須與現有首頁、恐龍故事書、水族館故事書、`/explore` 與 `/compare` 一致。
- **DR-002**: 題目與選項必須短而清楚，適合 5-10 歲孩童閱讀。
- **DR-003**: 答案狀態不得只靠顏色表示，必須搭配文字或狀態圖示。
- **DR-004**: 每次只顯示一題主要題目，避免同時顯示長題目列表造成閱讀負擔。
- **DR-005**: 亮色、深色與跟隨系統模式下都必須維持可讀文字、足夠對比與可見焦點。
- **DR-006**: 問答核心流程不得依賴拖曳、限時或需要精細手勢的互動。

### 非功能需求

- **NFR-001**: `/quiz` 頁面主要內容在完整資料情境下應於一般本機開發環境 1 秒內完成 server-rendered HTML 輸出。
- **NFR-002**: 答案提交、回饋顯示與下一題更新應在使用者操作後 1 秒內反映於目前頁面。
- **NFR-003**: 問答頁主要互動在 375px、768px、1366px 代表寬度下不得水平溢出或重疊。
- **NFR-004**: 問答功能不得改變瀏覽器 history、scroll position、語言偏好或主題偏好，除非使用者點擊一般導覽連結。
- **NFR-005**: 問答功能不得要求 JavaScript-only routing；所有故事導覽使用一般 anchor link。
- **NFR-006**: 題庫載入失敗、題目不足、未選答案、答對、答錯與無相關故事訊息必須使用 5-10 歲孩童能理解的友善文字。
- **NFR-007**: 題庫驗證失敗或來源載入失敗時必須使用 `ILogger<T>` 記錄，不得向使用者暴露內部例外細節、檔案路徑或敏感資訊。

## 關鍵實體 *(include if feature involves data)*

- **問答題目**: 一筆人工維護題目，包含 question id、範圍、難度、雙語題幹、雙語選項、正確選項、雙語回饋、雙語解釋與相關故事引用。
- **答案選項**: 題目中的一個可選答案，包含 option id、雙語文字與排序值。
- **問答範圍**: 使用者選擇的題目集合，目前包含全部故事、恐龍與水族館。
- **作答狀態**: 目前頁面中使用者選擇的題目、選項與回饋狀態；只存在於 `/quiz` 頁面互動生命週期。
- **題庫 snapshot**: 問答頁使用的唯讀投影，包含有效題目、範圍統計、來源狀態、相關故事連結與友善提示。

## 成功標準 *(mandatory)*

### 可量測結果

- **SC-001**: 使用者能在 5 秒內從首頁或 `/explore` 找到並進入問答挑戰。
- **SC-002**: 完整資料情境下，使用者能選擇全部故事、恐龍或水族館範圍並看到一題有效題目。
- **SC-003**: 使用者提交答案後，正確或錯誤回饋與解釋在 1 秒內顯示。
- **SC-004**: 100% 有效題目都至少有一個可開啟的既有故事詳情頁連結。
- **SC-005**: 在繁體中文與英文下，問答頁不得出現空白題目、空白選項、空白回饋或空白解釋。
- **SC-006**: 在 375px、768px 與 1366px 代表寬度下，問答頁不得水平溢出或出現文字與控制項重疊。
- **SC-007**: 鍵盤操作驗收中，範圍選擇、答案選項、提交、下一題與故事連結 100% 可聚焦並可啟用。
- **SC-008**: 題庫或 catalog 部分不可用時，問答頁顯示可用題目或友善錯誤狀態，不呈現空白頁。
- **SC-009**: 問答功能不得將作答結果寫入 URL、history、localStorage、sessionStorage、cookie 或伺服器端狀態。

## 開發建議

### 建議實作切片

1. 新增題庫資料模型、Options、validator 與 catalog service，從靜態 JSON 建立有效題目 snapshot。
2. 題庫 validator 驗證題目欄位、選項數量、唯一 id、單一正確答案、雙語 fallback 與相關故事引用。
3. 在 DI 註冊問答 service，並加入資料載入錯誤與引用驗證 logging。
4. 新增 `/quiz` Razor Page 與 PageModel，server-render 題庫範圍、有效題目、來源狀態與初始友善提示。
5. 新增 `quiz.css` 與 `quiz.js`，處理範圍選擇、單題顯示、答案提交、回饋、下一題與雙語狀態。
6. 在首頁或 `/explore` 加入「問答挑戰」一般 anchor 入口。
7. 補單元測試、整合測試與 JavaScript contract 測試，特別覆蓋題庫驗證、範圍篩選、相關故事連結與不保存作答狀態。
8. 依本規格進行手動驗收，確認 mobile/tablet/desktop、鍵盤、雙語、深色模式與題庫失敗情境。

## 假設

- 初始問答只使用既有恐龍與水族館內容，不新增第三本故事書。
- 題目由專案維護者人工撰寫與審核，不由 AI 或外部服務自動生成。
- 問答分數與作答結果不保存；若未來要把答題成果接入護照徽章，需另行更新護照與問答規格。
- 預設語言為繁體中文 `zh-TW`，英文為第二語言。
- 問答頁只做站內內容複習，不搜尋外部網站或百科資料。
