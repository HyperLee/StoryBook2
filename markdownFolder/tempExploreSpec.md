# Feature Specification Draft: 全站探索與分類搜尋

## 功能目標

建立一個全站探索入口，讓使用者可以一次搜尋與瀏覽目前網站中的恐龍介紹與水族館動物故事書內容。使用者不需要先判斷內容屬於哪一本故事書，就能透過關鍵字、分類、食性、生活環境或時期快速找到想看的自然朋友。

本功能延續 StoryBook 兒童友善、雙語、可直接連結、一般 anchor 導覽與深色模式一致套用的既有規則，不新增登入、資料庫、外部百科 API、即時翻譯或 JavaScript-only router。

## User Scenarios & Testing

### User Story 1 - 從首頁進入全站探索頁 (Priority: P1)

作為一位 5-10 歲孩童或陪讀家長，我希望能從首頁進入「探索全部故事」頁面，一次看到恐龍與水族館動物內容，讓我能快速開始找想看的故事。

**Why this priority**: 這是跨故事書探索的核心入口；若沒有清楚入口，使用者仍需要分別進入恐龍或水族館頁面搜尋。

**Independent Test**: 從首頁點擊「探索全部故事」入口後，確認進入探索頁，並看到恐龍與水族館內容卡片。

**Acceptance Scenarios**:

1. **Given** 使用者位於首頁，**When** 點擊「探索全部故事」，**Then** 進入全站探索頁。
2. **Given** 使用者進入探索頁，**When** 頁面載入完成，**Then** 顯示恐龍與水族館兩類內容。
3. **Given** 使用者查看任一探索結果，**When** 點擊結果卡片，**Then** 進入原本對應的詳情頁，例如 `/dinosaurs/{slug}` 或 `/aquarium/{slug}`。

---

### User Story 2 - 跨故事書搜尋內容 (Priority: P1)

作為一位使用者，我希望能在同一個搜尋框輸入名稱、食性、生活環境、生活時期或地點，找到恐龍與水族館中的相關內容。

**Why this priority**: 目前恐龍與水族館各自有搜尋；跨故事書搜尋能降低使用者找內容的成本。

**Independent Test**: 在探索頁輸入「草食」、「海水」、「北美洲」或英文關鍵字，確認結果可同時涵蓋不同故事書來源。

**Acceptance Scenarios**:

1. **Given** 使用者在探索頁，**When** 輸入「暴龍」，**Then** 顯示暴龍結果。
2. **Given** 使用者在探索頁，**When** 輸入「海水」，**Then** 顯示符合水族館生活環境或分類的結果。
3. **Given** 使用者輸入英文關鍵字，**When** 搜尋內容存在英文版本，**Then** 搜尋不因大小寫不同而漏掉結果。
4. **Given** 使用者輸入沒有符合內容的關鍵字，**When** 結果更新，**Then** 顯示孩童能理解的無結果提示。
5. **Given** 使用者清除搜尋，**When** 搜尋框變空，**Then** 恢復顯示完整可探索集合。

---

### User Story 3 - 用分類快速篩選故事內容 (Priority: P2)

作為一位家長或孩子，我希望能用簡單分類篩選內容，例如「恐龍」、「水族館」、「草食」、「肉食」、「淡水」、「海水」、「白堊紀」，讓我不用輸入文字也能探索主題。

**Why this priority**: 分類能幫助兒童用點選方式探索，但基本搜尋已能完成主要任務，因此排在 P2。

**Independent Test**: 點選分類後，結果只顯示符合該分類的內容，並可清除分類回到全部內容。

**Acceptance Scenarios**:

1. **Given** 使用者在探索頁，**When** 選擇「恐龍」，**Then** 只顯示恐龍故事書內容。
2. **Given** 使用者在探索頁，**When** 選擇「水族館」，**Then** 只顯示水族館故事書內容。
3. **Given** 使用者選擇食性或生活區域分類，**When** 結果更新，**Then** 顯示符合該分類的內容。
4. **Given** 使用者已套用分類，**When** 清除分類，**Then** 恢復完整結果。

---

### User Story 4 - 保持雙語與主題一致 (Priority: P2)

作為一位使用者，我希望探索頁和既有故事書一樣支援繁體中文與英文，也會套用目前的亮色、深色或跟隨系統主題。

**Why this priority**: 探索頁是跨故事書入口，必須與既有語言與主題體驗一致，避免使用者在頁面之間感到不連續。

**Independent Test**: 切換語言或主題後進入探索頁，確認頁面文字、結果摘要、提示訊息與卡片都使用正確語言與有效主題。

**Acceptance Scenarios**:

1. **Given** 使用者已選擇英文，**When** 進入探索頁，**Then** 探索頁主要文字與結果內容以英文顯示。
2. **Given** 使用者已選擇繁體中文，**When** 進入探索頁，**Then** 探索頁主要文字與結果內容以繁體中文顯示。
3. **Given** 使用者已選擇深色模式，**When** 進入探索頁，**Then** 探索頁套用深色外觀且不顯示新的主題 selector。
4. **Given** 某欄位缺少所選語言內容，**When** 顯示探索結果，**Then** 回退繁體中文且不得顯示空白標題或空白摘要。

## Edge Cases

- 搜尋字串為空白、只有標點或正規化後過短時，顯示友善提示並保留可探索內容。
- 搜尋與分類同時套用時，結果必須同時符合搜尋字串與分類條件。
- 沒有任何結果時，顯示可理解的提示，並提供清除搜尋與清除分類的動作。
- 來源 catalog 載入失敗時，探索頁不得空白；若只部分來源可用，需顯示可用內容與友善提示。
- 點擊探索結果必須前往既有 canonical route，不建立新的重複詳情頁。
- 語言偏好無效時回退 `zh-TW`；主題偏好無效時依既有主題規則回退 `system`。

## Requirements

### Functional Requirements

- **FR-001**: 系統 MUST 在首頁提供「探索全部故事」入口。
- **FR-002**: 系統 MUST 提供全站探索頁，顯示恐龍與水族館內容集合。
- **FR-003**: 系統 MUST 讓每個探索結果連回既有 canonical detail route。
- **FR-004**: 系統 MUST 提供跨故事書搜尋框。
- **FR-005**: 搜尋範圍 MUST 包含名稱、分類、食性、生活環境、生活時期、發現地點、摘要與 search keywords。
- **FR-006**: 搜尋 MUST 支援繁體中文與英文內容，英文搜尋忽略大小寫。
- **FR-007**: 搜尋輸入變更後 MUST 在 1 秒內更新結果、過短提示或無結果提示。
- **FR-008**: 系統 MUST 提供可點選分類篩選，至少包含來源類型：恐龍、水族館。
- **FR-009**: 系統 SHOULD 提供內容屬性分類，例如食性、生活區域、生活時期或地點。
- **FR-010**: 使用者 MUST 能清除搜尋與分類，回到完整探索集合。
- **FR-011**: 探索頁 MUST 支援 `storybook.language` 語言偏好與繁體中文 fallback。
- **FR-012**: 探索頁 MUST 套用既有 `storybook.theme` 有效主題，但不得提供新的主題 selector。
- **FR-013**: 探索結果卡片 MUST 顯示來源、名稱、短摘要、主要圖片或可辨識視覺、以及前往詳情頁動作。
- **FR-014**: 所有互動控制項 MUST 可鍵盤操作，有 accessible name 與可見焦點。
- **FR-015**: 本功能 MUST 不新增登入、資料庫、外部 API、即時翻譯、後台管理或 JavaScript-only router。

### Design Requirements

- **DR-001**: 探索頁視覺風格 MUST 與現有首頁、恐龍故事書與水族館故事書一致。
- **DR-002**: 搜尋框、分類篩選與清除控制 MUST 對 5-10 歲孩童足夠明顯且容易操作。
- **DR-003**: 探索結果 MUST 避免資訊過載，每張卡片只顯示足以辨識內容的重點資訊。
- **DR-004**: 探索頁 MUST 在亮色、深色與跟隨系統模式下維持可讀性與可見焦點。

### Non-Functional Requirements

- **NFR-001**: 探索頁主要互動在 375px、768px、1366px 寬度下不得水平溢出或重疊。
- **NFR-002**: 搜尋與分類更新不得改變瀏覽器 history、scroll position、語言偏好或主題偏好。
- **NFR-003**: 探索頁不得要求 JavaScript-only routing；所有結果導覽使用一般 anchor link。
- **NFR-004**: 找不到內容、搜尋無結果與資料載入失敗訊息必須使用 5-10 歲孩童能理解的友善文字。

## Key Entities

- **ExplorationItem**: 一筆可探索內容，來源於恐龍或水族館 catalog，包含 sourceType、slug、displayName、summary、image、route、searchText 與 filter facets。
- **ExplorationFilter**: 使用者選擇的篩選條件，例如來源類型、食性、生活區域、生活時期。
- **ExplorationSearchState**: 搜尋字串、語言、已選分類、符合結果與目前提示狀態。
- **SourceStorybook**: 內容來源類型，目前包含 dinosaurs 與 aquarium。

## Success Criteria

- **SC-001**: 95% 的使用者能在 5 秒內從首頁找到並進入探索頁。
- **SC-002**: 使用者能在 5 秒內用搜尋找到已知存在的恐龍或水族館動物。
- **SC-003**: 搜尋或分類變更後，結果或提示在 1 秒內更新。
- **SC-004**: 100% 的探索結果連結可直接開啟對應既有詳情頁。
- **SC-005**: 在 `zh-TW` 與 `en` 下，探索頁不得出現空白標題、空白摘要或沒有說明的圖片區塊。
- **SC-006**: 在 375px、768px、1366px 寬度下，探索頁不得水平溢出或出現文字與控制項重疊。
- **SC-007**: 鍵盤操作驗收中，搜尋框、分類篩選、清除控制與結果連結都可聚焦與啟用。

## Assumptions

- 本功能重用既有恐龍與水族館 catalog，不建立新的內容資料來源。
- 初始範圍只整合目前兩本故事書；未來新增第三本故事書時，可再擴充探索來源。
- 預設語言為繁體中文 `zh-TW`，英文為第二語言。
- 探索頁只做站內內容探索，不搜尋外部網站或百科資料。
- 搜尋與分類狀態不需要跨頁保存；重新進入探索頁時可回到完整集合。
