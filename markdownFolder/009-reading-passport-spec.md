# 功能規格: 小小探險護照

**Feature Branch**: `009-reading-passport`

**Created**: 2026-05-30

**Status**: Draft

**Input**: User description: "根據既有 001-005 規格與目前專案，新增下一階段可擴充功能規格至 markdownFolder。選定項目為學習閉環中的小小探險護照 MVP。"

## 功能目標

建立「小小探險護照」功能，讓孩子在閱讀恐龍或水族館故事詳情頁後，可以主動點選「我讀完了」或等價控制，把該故事朋友加入同一瀏覽器的本機閱讀護照。使用者可在 `/passport` 查看已讀故事數量、總數、已讀清單與簡單徽章，增加完成感與回訪動機。

本功能只使用瀏覽器 localStorage 保存非敏感、非個人化的本機閱讀狀態，不新增登入、資料庫、伺服器端紀錄、跨裝置同步、排行榜、個人資料蒐集或外部 API。護照狀態不得包含姓名、年齡、班級、學校、自由輸入文字或任何可識別個人的資料。

## 選擇此項目的理由

小小探險護照能把一次性閱讀轉成可累積的學習體驗，原因如下：

- 能讓孩子明確知道自己已讀過哪些故事朋友，提升完成感。
- 可重用現有恐龍與水族館 canonical detail route，不需要新增內容頁。
- 技術範圍可控，只需 localStorage 與 client-side module，不需要登入或資料庫。
- 可與未來學習旅程、問答挑戰、列印活動單自然整合。
- 清除本機資料的邊界清楚，能避免影響既有語言與主題偏好。

## 使用者情境與測試 *(mandatory)*

### 使用者故事 1 - 在故事頁標記讀完 (Priority: P1)

作為一位孩子，我希望在讀完一篇恐龍或水族館故事後，可以按下「我讀完了」，讓護照記住我已經認識這位故事朋友。

**優先級理由**: 主動完成標記是護照功能的核心資料來源；沒有它，護照頁無法呈現有意義的進度。

**獨立測試**: 開啟任一恐龍或水族館詳情頁，點選「我讀完了」，重新整理後確認該頁仍顯示已完成狀態。

**安全/資料影響**: 只寫入 localStorage 中的非敏感故事項目識別，不提交到伺服器，不包含個人資料。

**回滾方式**: 移除詳情頁完成控制與護照 JavaScript module；既有故事內容、搜尋、上一頁/下一頁與語言/主題偏好不受影響。

**驗收情境**:

1. **Given** 使用者位於 `/dinosaurs/triceratops`，**When** 點選「我讀完了」，**Then** 系統把 `dinosaurs:triceratops` 記錄到護照狀態。
2. **Given** 使用者位於 `/aquarium/sea-turtle`，**When** 點選「我讀完了」，**Then** 系統把 `aquarium:sea-turtle` 記錄到護照狀態。
3. **Given** 某故事已被標記讀完，**When** 使用者重新整理同一詳情頁，**Then** 系統顯示該故事已讀完的友善狀態。
4. **Given** 使用者重複點選「我讀完了」，**When** 護照狀態更新，**Then** 系統不得重複計算同一故事項目。

---

### 使用者故事 2 - 查看我的探險護照 (Priority: P1)

作為一位孩子或陪讀家長，我希望能開啟 `/passport` 查看已讀故事數量、總故事數與已讀清單，讓我知道目前探索進度。

**優先級理由**: 護照頁是完成狀態的主要回饋畫面，能讓孩子看見累積成果。

**獨立測試**: 標記至少兩篇故事讀完後進入 `/passport`，確認頁面顯示已讀數量、總數、已讀故事清單與前往故事詳情頁的連結。

**安全/資料影響**: 護照頁只讀取 localStorage 與公開 catalog，不向伺服器傳送護照狀態。

**回滾方式**: 移除 `/passport` Razor Page 與首頁入口；既有 localStorage key 可由使用者或清除控制移除。

**驗收情境**:

1. **Given** 使用者已標記 2 篇故事讀完，**When** 進入 `/passport`，**Then** 系統顯示已讀 2 筆與總故事數。
2. **Given** 護照中包含恐龍故事，**When** 護照頁顯示已讀清單，**Then** 該項目提供前往 `/dinosaurs/{slug}` 的一般 anchor link。
3. **Given** 護照中包含水族館故事，**When** 護照頁顯示已讀清單，**Then** 該項目提供前往 `/aquarium/{slug}` 的一般 anchor link。
4. **Given** 使用者尚未標記任何故事，**When** 進入 `/passport`，**Then** 系統顯示鼓勵開始閱讀的友善空狀態與前往故事入口的動作。

### 使用者故事 3 - 清除本機護照資料 (Priority: P1)

作為一位家長或共用裝置使用者，我希望可以清除這台瀏覽器中的護照紀錄，且不要影響語言與主題偏好。

**優先級理由**: localStorage 狀態必須可由使用者控制，尤其此專案面向兒童與共用裝置。

**獨立測試**: 在 `/passport` 點選清除護照並確認後，檢查已讀清單歸零，但 `storybook.language` 與 `storybook.theme` 保持原值。

**安全/資料影響**: 清除動作只刪除 `storybook.readingPassport`，不得使用大量清除 localStorage、不得刪除其他同站偏好。

**回滾方式**: 移除清除控制後，護照仍可顯示狀態；若整個護照功能移除，只需停止讀寫 `storybook.readingPassport`。

**驗收情境**:

1. **Given** 護照已有已讀項目，**When** 使用者選擇清除護照，**Then** 系統顯示清楚確認文字。
2. **Given** 使用者確認清除，**When** 清除完成，**Then** 系統移除 `storybook.readingPassport` 並顯示空護照狀態。
3. **Given** 使用者已設定英文語言偏好，**When** 清除護照完成，**Then** `storybook.language` 不得被刪除或改寫。
4. **Given** 使用者已設定深色主題偏好，**When** 清除護照完成，**Then** `storybook.theme` 不得被刪除或改寫。

---

### 使用者故事 4 - localStorage 不可用時仍可閱讀 (Priority: P2)

作為一位使用者，我希望即使瀏覽器限制 localStorage，故事頁仍能正常閱讀，護照功能也能用友善文字說明暫時不能保存。

**優先級理由**: 本功能依賴瀏覽器本機儲存；不可用時必須降級，不得破壞既有故事閱讀流程。

**獨立測試**: 模擬 localStorage 讀寫失敗後進入故事詳情頁與 `/passport`，確認故事內容仍完整顯示，並以友善提示說明護照暫時不能保存。

**安全/資料影響**: 不得因 localStorage 失敗而改用 cookie、伺服器端狀態或 URL query 保存護照資料。

**回滾方式**: 移除護照 script 後，故事頁恢復純閱讀模式；既有功能不受影響。

**驗收情境**:

1. **Given** localStorage 不可讀取，**When** 使用者開啟故事詳情頁，**Then** 故事內容、圖片、搜尋與導覽仍可正常使用。
2. **Given** localStorage 不可寫入，**When** 使用者點選「我讀完了」，**Then** 系統顯示護照暫時不能保存的友善提示。
3. **Given** localStorage 不可用，**When** 使用者進入 `/passport`，**Then** 系統顯示不能讀取本機護照的友善狀態與回到故事入口的動作。
4. **Given** localStorage 不可用，**When** 系統處理護照狀態，**Then** 不得把護照資料寫入 URL、cookie 或伺服器端狀態。

---

### 使用者故事 5 - 保持雙語、主題與可及性一致 (Priority: P2)

作為一位使用者，我希望護照頁和既有故事書一樣支援繁體中文與英文，並套用目前主題，讓完成狀態與閱讀體驗一致。

**優先級理由**: 護照跨恐龍與水族館頁面，必須遵守既有語言、主題與可及性規則。

**獨立測試**: 切換語言或主題後進入故事詳情頁與 `/passport`，確認完成控制、護照頁文字、清除確認、提示訊息、焦點狀態與操作目標都符合目前語言與有效主題。

**安全/資料影響**: 語言與主題沿用既有 `localStorage` 偏好；護照頁不得新增 theme selector、server-side preference endpoint、cookie、帳號同步或新的資料保存。

**回滾方式**: 移除護照專用雙語與樣式後，頁面可退回共用 layout；既有語言切換與主題服務不受影響。

**驗收情境**:

1. **Given** 使用者已選擇英文，**When** 進入 `/passport`，**Then** 護照頁主要文字、狀態訊息與已讀項目以英文顯示。
2. **Given** 使用者已選擇繁體中文，**When** 進入 `/passport`，**Then** 護照頁主要文字與狀態訊息以繁體中文顯示。
3. **Given** 使用者已選擇深色模式，**When** 進入 `/passport`，**Then** 頁面套用深色外觀且不顯示新的主題選擇控制項。
4. **Given** 使用者使用鍵盤操作護照控制，**When** 焦點移動到「我讀完了」、護照入口、清除控制或故事連結，**Then** 焦點清楚可見且控制項可被啟用。
5. **Given** 某已讀項目缺少所選語言內容，**When** 護照頁顯示清單，**Then** 系統回退繁體中文且不得顯示空白標題或空白摘要。

## 邊界情況

- 使用者尚未讀完任何故事時，護照頁顯示鼓勵開始探索的友善空狀態。
- localStorage key 缺漏、內容不是 JSON、JSON 結構錯誤或版本不支援時，系統回退空護照狀態並顯示友善提示。
- localStorage 讀取或寫入拋出例外時，故事頁仍可閱讀，護照狀態不保存。
- 護照資料包含不存在的 source type 或 slug 時，系統不得顯示失效連結，需略過該項目或顯示友善不可用提示。
- 同一故事項目被標記多次時，只能計算一次。
- 使用者清除護照時，只能移除 `storybook.readingPassport`。
- 語言偏好無效、遺失或翻譯缺漏時，系統回退繁體中文。
- 主題偏好無效、遺失或無法判斷時，頁面依既有整站主題規則採用安全外觀。
- 護照不得自動把開啟故事頁視為讀完，必須由使用者明確動作觸發。
- 護照不得保存姓名、年齡、班級、學校、自由輸入文字、精確時間線或其他可識別個人的資料。

## 需求 *(mandatory)*

### 範圍邊界

- 本階段包含首頁或共用導覽中的護照入口、canonical route `/passport`、恐龍詳情頁與水族館詳情頁的完成閱讀控制、已讀清單、已讀數量、簡單徽章、清除本機護照資料控制與友善提示。
- 本階段只整合目前既有的恐龍故事書與水族館故事書內容。
- 本階段可新增 client-side 護照 module 與必要的 server-rendered catalog projection，但不得改動 `StoryBook/Data/dinosaurs.json` 或 `StoryBook/Data/aquarium.json` 的既有 schema。
- 本階段不包含登入、跨裝置同步、伺服器端學習紀錄、排行榜、社群分享、個人化推薦、後台管理、外部 API、即時翻譯或使用者上傳內容。
- 本階段不將護照狀態寫入 URL query、browser history、sessionStorage、cookie、伺服器端狀態或任何外部服務。
- 本階段不改變既有恐龍、水族館、`/explore`、`/compare`、語言或主題規則。

### 功能需求

- **FR-001**: 系統必須在首頁或共用導覽至少一處提供清楚可辨識的「我的探險護照」入口。
- **FR-002**: 系統必須在 canonical route `/passport` 提供探險護照頁面。
- **FR-003**: 系統必須在恐龍詳情頁與水族館詳情頁提供「我讀完了」或等價完成閱讀控制。
- **FR-004**: 完成閱讀控制必須由使用者明確操作觸發，不得因頁面載入、捲動、停留時間或圖片開啟自動標記完成。
- **FR-005**: 系統必須使用 localStorage key `storybook.readingPassport` 保存本機護照狀態。
- **FR-006**: 護照狀態必須使用穩定故事項目識別，例如 `dinosaurs:tyrannosaurus-rex` 或 `aquarium:clownfish`。
- **FR-007**: 系統必須避免同一故事項目重複計算。
- **FR-008**: 護照頁必須顯示已讀故事數量、總故事數、已讀清單與前往既有詳情頁的連結。
- **FR-009**: 護照頁必須顯示至少一種簡單徽章或里程碑，例如讀完第一篇、讀完 3 篇、讀完一整本故事書。
- **FR-010**: 徽章或里程碑必須由本機已讀項目 deterministic 推導，不得使用伺服器端狀態或外部服務。
- **FR-011**: 護照中的恐龍故事項目連結必須前往 `/dinosaurs/{slug}`。
- **FR-012**: 護照中的水族館故事項目連結必須前往 `/aquarium/{slug}`。
- **FR-013**: 使用者必須能清除本機護照資料，並在清除前看到清楚確認文字。
- **FR-014**: 清除護照只能刪除 `storybook.readingPassport`，不得刪除或改寫 `storybook.language`、`storybook.theme` 或其他 storage key。
- **FR-015**: localStorage 不可讀取、不可寫入或資料格式無效時，系統必須顯示友善提示並維持故事閱讀可用。
- **FR-016**: 護照頁與完成控制必須支援既有網站語言偏好，並在偏好無效或內容缺漏時回退繁體中文。
- **FR-017**: 護照頁必須套用既有整站有效主題，但不得提供新的主題選擇控制項。
- **FR-018**: 主題變更只能影響護照頁視覺呈現與控制項狀態，不得改變故事內容、護照資料、語言偏好、導覽歷史或頁面捲動位置。
- **FR-019**: 所有使用者可見提示、空狀態、確認文字與錯誤訊息必須使用 5-10 歲孩童能理解的友善文字。
- **FR-020**: 所有互動控制項必須可用滑鼠、觸控與鍵盤操作，並具備可理解的 accessible name 與可見焦點。
- **FR-021**: 護照頁在手機、平板與桌面代表寬度下不得出現水平溢出、文字重疊或控制項互相遮擋。
- **FR-022**: 護照頁不得顯示空白標題、空白摘要、空白來源標籤或失效連結。
- **FR-023**: 護照功能不得要求使用者登入、不得搜尋外部網站、不得新增即時翻譯、不得新增外部內容來源。
- **FR-024**: 護照狀態不得包含姓名、年齡、班級、學校、自由輸入文字或其他可識別個人的資料。
- **FR-025**: 護照頁預設排序必須先依來源故事書排序，再保留各來源既有 catalog 順序。

### 設計需求

- **DR-001**: 護照頁視覺風格必須與現有首頁、恐龍故事書、水族館故事書、`/explore` 與 `/compare` 一致。
- **DR-002**: 完成閱讀控制必須放在故事詳情頁中不干擾主要閱讀的位置，且標籤需讓 5-10 歲孩童能理解。
- **DR-003**: 已讀狀態不得只靠顏色表示，必須有文字、圖示或狀態標籤。
- **DR-004**: 清除護照控制必須與主要閱讀入口區隔，避免兒童誤觸。
- **DR-005**: 亮色、深色與跟隨系統模式下都必須維持可讀文字、足夠對比與可見焦點。

### 非功能需求

- **NFR-001**: `/passport` 頁面主要內容在完整資料情境下應於一般本機開發環境 1 秒內完成 server-rendered HTML 輸出。
- **NFR-002**: 完成閱讀控制、護照狀態更新與清除結果應在使用者操作後 1 秒內反映於目前頁面。
- **NFR-003**: 護照頁主要互動在 375px、768px、1366px 代表寬度下不得水平溢出或重疊。
- **NFR-004**: 護照功能不得改變瀏覽器 history、scroll position、語言偏好或主題偏好，除非使用者點擊一般導覽連結。
- **NFR-005**: 護照功能不得要求 JavaScript-only routing；所有故事導覽使用一般 anchor link。
- **NFR-006**: localStorage 讀寫失敗、資料格式錯誤、空護照與清除結果訊息必須使用 5-10 歲孩童能理解的友善文字。
- **NFR-007**: localStorage 解析失敗或來源載入失敗時必須使用 `ILogger<T>` 或 client-side 安全訊號記錄可診斷資訊，不得向使用者暴露內部例外細節、檔案路徑或敏感資訊。

## 關鍵實體 *(include if feature involves data)*

- **閱讀護照狀態**: localStorage 中的本機資料，包含 schema version、已讀故事項目識別集合與必要的非敏感更新資訊。
- **已讀故事項目**: 由來源類型與 slug 組成的穩定識別，對應既有恐龍或水族館詳情頁。
- **護照徽章**: 根據已讀故事項目 deterministic 推導的里程碑，例如第一篇故事、三位故事朋友、恐龍故事書進度或水族館故事書進度。
- **護照 snapshot**: 護照頁使用的唯讀投影，結合本機已讀狀態與 server-rendered catalog 資訊，包含已讀數、總數、清單、徽章與友善提示。

## 成功標準 *(mandatory)*

### 可量測結果

- **SC-001**: 使用者能在任一恐龍或水族館詳情頁找到並操作完成閱讀控制。
- **SC-002**: 使用者完成閱讀後，重新整理同一瀏覽器頁面仍能看到該故事已讀完狀態。
- **SC-003**: 使用者能在 5 秒內從首頁或共用導覽找到並進入 `/passport`。
- **SC-004**: `/passport` 能正確顯示已讀故事數量、總故事數、已讀清單與至少一種徽章或里程碑。
- **SC-005**: 清除護照後，`storybook.readingPassport` 被移除，且 `storybook.language` 與 `storybook.theme` 保持不變。
- **SC-006**: localStorage 不可用時，100% 既有故事閱讀流程仍可使用，且頁面顯示友善降級提示。
- **SC-007**: 在繁體中文與英文下，護照頁不得出現空白標題、空白摘要、空白來源標籤或空白狀態文字。
- **SC-008**: 在 375px、768px 與 1366px 代表寬度下，護照頁不得水平溢出或出現文字與控制項重疊。
- **SC-009**: 鍵盤操作驗收中，完成閱讀控制、護照入口、清除控制與故事連結 100% 可聚焦並可啟用。

## 開發建議

### 建議實作切片

1. 定義護照 localStorage schema、允許值與資料遷移策略；v1 僅保存已讀 item ids。
2. 新增 client-side 護照 module，集中處理讀取、寫入、去重、清除與 localStorage 失敗 fallback。
3. 在恐龍與水族館詳情頁加入完成閱讀控制與必要 data attributes。
4. 新增 `/passport` Razor Page 與 PageModel，server-render 可用 catalog、總數、初始空狀態與護照頁文案。
5. 新增 `passport.css` 與 `passport.js`，顯示本機已讀狀態、徽章、清除確認與友善提示。
6. 在首頁或共用導覽加入「我的探險護照」一般 anchor 入口。
7. 補單元測試、整合測試與 JavaScript contract 測試，特別覆蓋 storage key、去重、清除與錯誤格式。
8. 依本規格進行手動驗收，確認 mobile/tablet/desktop、鍵盤、雙語、深色模式與 localStorage blocked 情境。

## 假設

- 護照只保存同一瀏覽器的本機狀態，不跨裝置同步。
- 護照不保存個人資料，也不宣稱可追蹤真實學習成效。
- 完成閱讀必須由使用者主動按下控制，不自動偵測閱讀完成。
- 預設語言為繁體中文 `zh-TW`，英文為第二語言。
- 初始範圍只整合既有恐龍與水族館詳情頁；旅程完成與問答成績可在未來規格另行整合。
